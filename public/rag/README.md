# public/rag/ — precomputed RAG artifacts

Static JSON files generated by `rag/precompute.mjs` against the
populated Pinecone civil-rights index. Netlify serves them at the
site root (`https://<host>/rag/...`); React components fetch them
directly without any Function call.

## Files

| Path | Generator flag | Consumed by | Purpose |
|---|---|---|---|
| `related/entry-<N>.json` | `--feature related` | `<RelatedPassages>` | Per-entry top-5 related passages from other interviews (per-chunk + aggregated summary). One file per corpus entry. |
| `centroids.json` | `--feature centroids` | (intermediate; substrate for constellation) | Per-entry mean embedding sampled across the entry's chunks. |
| `constellation.json` | `--feature constellation` | `<Constellation>` | 2D PCA projection of the entry centroids for the scatter visualization. |

## Regenerating

```bash
node --env-file=rag/.env.local rag/precompute.mjs
# or one feature at a time:
node --env-file=rag/.env.local rag/precompute.mjs --feature related
node --env-file=rag/.env.local rag/precompute.mjs --feature constellation
```

Run after any change to the Pinecone index (re-ingests, manifest edits,
new entries). The output is gitignored at the directory level but the
individual `*.json` files are committed so Netlify deploys are
deterministic without needing Pinecone credentials at build time.

The same generator + UI components port verbatim to the worldthought.com
RAG layer by pointing `PINECONE_HOST` / `PINECONE_INDEX` at that
project's index. See `rag/INTERACTIVE_FEATURES_DESIGN.md` for the
full portability pattern.
